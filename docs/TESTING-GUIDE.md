# 学院综合服务平台 — 测试使用说明

> 这份文档面向**第一次接触本系统的测试人员 / 评审老师 / 同班同学**。
> 不需要写代码，按章节走一遍就能体验全部核心功能并帮我们发现问题。
>
> 部署地址（内网）：**http://10.10.0.27**
> 后端 API：http://10.10.0.27/api
> 测试时间窗：内部演示及答辩期间 24h 可用

---

## 一、能做什么

系统覆盖学院学生日常**五个高频场景**：

| 模块 | 谁用 | 主要操作 |
|---|---|---|
| 智能问答（RAG） | 学生 | 用自然语言提问政策制度，系统检索政策文档抽取式回答 |
| 政策文档 / 办公模板 | 学生 | 浏览/下载政策文件 + 党员证明等办公模板 |
| 党团流程 | 学生 / 教师 | 学生查看个人入党/入团进度；教师维护流程模板和实例 |
| 电子证明审批 | 学生 / 教师 | 学生按模板申请 → 教师审批 → 学生预览/下载自动生成的 PDF |
| 信息精准推送 | 教师 → 学生 | 教师按角色/年级/专业群发通知，可走站内 + 邮件，24h 内可撤回 |

## 三、PC 管理端

### 3.1 怎么打开

浏览器（Chrome / Edge / 火狐都行）访问：

> **http://10.10.0.27**

用 `admin` / `admin123` 登录。手机浏览器也能开，但页面是为桌面端设计的，建议用电脑。

### 3.2 关键测试路径

按"左侧菜单从上到下"的顺序走一遍：

#### 1) 数据概览
- 顶部 4 个数字卡（学生总数 / 用户数 / 待审批 / 进行中流程）
- 下面 3 张图：审批状态饼图 / 党团流程柱图 / 7 日趋势折线
- 右侧最新 5 条待审批列表

**测试点**：数字与实际数据是否一致？图表无数据时是否显示空状态而不是空骨架？

#### 2) 智能问答
- **知识库管理**：新增一条标准问答（如 "国奖申请时间" / "每年 10 月"），看是否能被学生端检索到
- **政策文档**：上传一份 PDF 政策文件 → 点击"重新索引" → 等 5-10 秒后让学生端提问相关内容，应能拿到来自 RAG 的抽取式回答
- **办公模板**：管理员视角能看到办公模板列表（doc_type=template）

#### 3) 党团流程
- 流程模板：查看入党流程（29 步）和入团流程（5 步），点编辑能动态增删步骤
- 流程实例：可以为某学生创建一个新实例，推进到下一步

#### 4) 审批管理
- **待审批**：列表显示 申请人姓名 / 学号 / 证明类型 / 提交时间。点"详情"看 formData。
  - 测试：点"通过" → 学生侧应能预览 PDF + 下载
  - 测试：点"驳回"填原因 → 学生侧应看到驳回原因
- **全部申请**：按状态筛选（草稿/待审批/已通过/已驳回/已撤回/已归档），状态彩色 tag 区分
  - 测试：已下载状态的申请，**禁止**撤回（按钮置灰）

#### 5) 学生信息
- 列表 + 筛选年级 / 专业
- 点详情 → 抽屉显示 学生基础信息 + 荣誉记录 + 党团流程 + 申请记录 4 块
- 可以"新增荣誉"

#### 6) 用户管理
- **下载导入模板**：点击得到 `.xlsx` 模板，列与后端约定一致
- **Excel 导入**：填几行学生信息后导入，结果弹窗会显示 成功/失败/错误明细
- **导出名单**：按当前筛选条件导出 `.xlsx`
- **设置角色**：把某学生临时升到 2 级，登录他的账号应能进管理端

#### 7) 系统管理 → 通知群发
- **新建群发**：填标题、正文、勾选标签（如"奖学金/实习"）、目标筛选 → "预览目标人数" → "确认群发"
  - 渠道选 system + email（如果配了 SMTP 真实邮件会发出去）
  - 测试：学生端是否立即在消息中心看到这条通知
- **广播历史**：点"撤回"，确认目标用户中**未读**的通知消失，**已读**的保留

#### 8) 操作日志（仅 1 级可见）
- 显示管理员的关键操作流水（通过/驳回/导入/群发等）

#### 扫码后第一次进入必须做一件事：**打开调试模式**

体验版的后端目前部署在**内网 IP `10.10.0.27` + HTTP**（生产域名 / HTTPS 答辩后再上）。微信客户端默认拒绝 HTTP 请求 → 不开调试会一直**网络错误**。

操作（每个测试者**只需做一次**）：

1. 微信扫上面的二维码进入小程序
2. 右上角点 **`···`**（三个点）
3. 找到 **"打开调试"** 开关 → 点开
4. **退出小程序**（左滑删除最近列表，或后退到微信）
5. **重新扫码**进入

完成后屏幕底部会出现绿色 `vConsole` 调试按钮，登录、所有功能都能正常用了。

> 没开调试就登录会报"网络错误"，不是 bug；按上述步骤打开即可。

#### 网络前提

体验版后端运行在 `10.10.0.27`（学院内网）。**测试时请：**

- 连**学院内网 WiFi** 或 VPN 进内网
- 验证方法：手机自带浏览器打开 `http://10.10.0.27/`，能看到管理端登录页则网络通

如果手机连的是 4G 或外网 WiFi，无论怎么调小程序都会报网络错误。

### 4.3 关键测试路径

按 TabBar 从左到右走：

#### 1) 首页
- 右上铃铛若有未读通知，应有小红点
- 中间三个快捷入口 + 下方四张大卡片（智能问答 / 文件模板 / 党团进度 / 我的申请）
- 最下"最新通知"显示 5 条

#### 2) 智能问答
- 输入框输入"国奖什么时候申请" / "入党流程多少步" / "请假流程怎么走"
- 应在 1-2 秒内返回带"依据 N"标注的抽取式回答，下方有来源链接

#### 3) 政策文档 / 办公模板
- Tab 切换 政策文档 / 办公模板
- 不同文件类型左侧色块不同色（PDF 红 / DOCX 蓝 / XLSX 绿等）
- 点 ↓ 下载，应弹出微信内嵌的 PDF/DOCX 预览
- 若某模板显示"待上线"徽章，点击会提示尚未上线（这是设计的占位）

#### 4) 党团进度
- 上半显示个人正在进行的流程（无则空状态）
- 下半两张卡片显示官方入党/入团流程模板
- 点入党模板 → 详情页 → 看到 29 步纵向时间线

#### 5) 我的申请
- 顶部统计：待审批 / 可下载 / 已归档 三个数字
- 列表显示申请卡片，标题是模板名（如"党员证明模板"）
- 已通过状态：有"预览"+"下载证明"两个按钮
- 待审批状态：有"撤回申请"按钮
- 已归档状态：只剩"锁定归档"灰色徽标
- **核心测试**：点底部"提交新申请" →
  1. 选一个模板（如 党员证明模板）
  2. 上方显示已自动填的 个人信息（姓名/学号/年级/专业/班级）
  3. 下方表单要求补充 身份证号 / 学历 / 入党日期 / 党支部 / 用途
  4. 提交后回到列表 → 状态变为"待审批"
  5. 让 admin 通过后 → 学生点"预览"应能在微信里打开 PDF，看到中文正确呈现

#### 6) 消息中心
- 横向标签筛选条
- 未读条目左侧深红色边框
- 点击条目自动标为已读
- 顶部"全部标记已读"按钮

#### 7) 个人中心
- 头像（CSS 人物剪影）+ 姓名 + 学号
- 三个快捷行：我的申请 / 党团进度 / 消息中心
- 联系方式：点邮箱"修改"，弹出输入框；改成无效格式（如 `xxx`）应被拒绝；输入空串保存，应回落默认 `学号@ruc.edu.cn`
- 我的荣誉：admin 给学生录荣誉后这里能看到

## 五、典型 E2E 测试场景

### 场景 1：学生申请党员证明 → 教师审批 → 学生下载

1. 小程序登 `20240001 / admin123`
2. 我的申请 → 提交新申请 → 选"党员证明模板"
3. 补充身份证号 / 学历 / 入党日期 / 党支部 / 用途 → 提交
4. 浏览器登管理员 → 审批管理 → 待审批 → 找到新申请 → 点"通过"
5. 小程序刷新"我的申请" → 该申请状态变为"已通过"
6. 点"预览" → 微信内嵌打开 PDF（含中文，模板格式套用学生信息）
7. 点"下载证明" → 弹确认 → 下载完毕，状态变为"已归档"
8. 之后再尝试"撤回"应被拒绝

### 场景 2：教师群发通知 → 学生收到

1. 浏览器登管理员 → 系统管理 → 通知群发
2. 标题 "测试通知"，正文 "请尽快办理学生证年检"
3. 标签选 "学籍"，目标筛选 角色=4级 / 专业=计算机科学
4. 预览目标人数 → 应显示 1 人
5. 提交群发
6. 小程序登 `20240001` → 消息中心 → 顶部出现新通知 + 未读边框
7. 标签筛选切到"学籍" → 只显示这条
8. 返回管理员 → 广播历史 → 找到这条 → 点"撤回"
9. 学生再刷新消息中心 → 该通知**消失**（前提：学生还没标已读）

### 场景 3：智能问答检索

1. 管理员上传一份 RUC 学籍管理规定 PDF → 重新索引
2. 学生小程序 → 智能问答 → 提问 "本科生最多可以读几年"
3. 应返回带"依据"标签的抽取式回答 + 来源文档链接

## 六、已知限制 / 不算 bug

| 现象 | 说明 |
|---|---|
| 政策文档列表里"请假条模板/活动预算表/班会简报模板"显示"待上线" | 模板源 docx 还没补，**故意**留的占位 |
| PDF 证明落款"中国人民大学信息学院党委 2026-05-XX"是当天日期 | 系统自动填生成日期，无法手改 |
| 申请提交后立刻在管理员端能看到 typeId/templateDocId 列空 | 已废弃旧 type_id 字段，新申请只用 template_doc_id |
| 邮件通知发出去 `emailSent=0` | 部署时 `MAIL_AUTH_CODE` 没填，自动降级为站内 `email_sim` 通知 |
| 智能问答偶尔回 "未在政策文件中找到明确依据" | 阈值卡得较严防误命中。可在 admin 端 → 知识库管理补充标准问答 |

  2) ...
【期望结果】...
