## 二、PC 管理端功能

进入后左侧菜单按角色动态显示，1 级看全部，2 级看大部分，3 级只能看本班数据。

### 2.1 数据概览

- 4 个统计卡：在校学生数 / 系统用户数 / 待审批申请数 / 进行中流程数
- 审批状态分布饼图（草稿 / 待审批 / 已通过 / 已驳回 / 已撤回 / 已归档）
- 党团流程模板分布柱图（每个模板下的实例数）
- 近 7 日活动趋势折线图（申请提交数 / 系统通知数）
- 最新 5 条待审批列表

### 2.2 智能问答

- **知识库管理**：维护「标准问题 → 标准答案」键值对，支持分类筛选、关键词检索、新增 / 编辑 / 删除
- **政策文档管理**：上传政策 PDF / DOCX / TXT，支持下载、删除、**向量入库**（点击后后端用 BGE-small-zh 嵌入模型切片 + 算向量写 `qa_document_chunk` 表，供学生端 RAG 检索）
- **办公模板管理**：管理 `doc_type=template` 的 docx 模板，学生端会在"我的申请"页用这些模板申请证明

### 2.3 党团流程

- **流程模板管理**：内置入党流程（29 步，按《发展党员工作程序》）和入团流程（5 步），支持动态增删步骤、每步可设预计天数（到期提醒任务会读这个字段）
- **学生流程实例管理**：为指定学生创建一个流程实例，支持推进到下一步 / 暂停 / **恢复（暂停后可继续）** / **删除** / 查看进度

### 2.4 审批管理

- **待审批列表**：显示申请编号、申请人姓名 + 学号、证明类型、当前审批层级、提交时间。点详情看学生填的表单内容；点通过 / 驳回触发状态机流转
- **全部申请列表**：按状态筛选（6 种），状态彩色 tag 区分；管理员可对「已通过未下载」的申请执行撤回
- **审批流程内置规则**：通过后 2 天内可撤回；学生下载证明后状态变为 `downloaded`，**任何修改操作被禁止**（状态机层做硬约束）

### 2.5 学生信息

- 学生列表涵盖**普通学生 + 学生骨干**（骨干本质也是学生），列表「身份」列用标签区分两者
- 筛选栏：身份（普通学生 / 学生骨干）、年级、专业、班级；**年级 / 专业 / 班级均为下拉选择**，选项来自库中真实存在的值（避免手输错字匹配不到），选中即筛
- 详情抽屉显示 4 块：基础信息、荣誉记录、党团流程、申请记录
- 支持新增 / 删除荣誉记录
- 敏感字段（身份证、生源地、户籍地）**AES 加密入库 + 前 3 后 4 留明文显示**
- 3 级骨干登录时只能看到本班学生（数据隔离）

### 2.6 用户管理

- 用户列表覆盖全部角色（院领导 / 老师 / 骨干 / 学生）；筛选栏支持**身份**（院领导 / 管理老师 / 学生骨干 / 普通学生）+ 年级 / 专业 / 班级，后三者为下拉选择
- **Excel 批量导入学生名单**：下载导入模板 → 填写 → 上传，结果弹窗显示成功 / 失败 / 错误明细
- **Excel 导出学生名单**：只导出**启用状态的学生（普通学生 + 学生骨干）**，不含老师 / 院领导 / 已禁用账号；导出文件含「身份」列区分两类。点导出会弹确认框列明本次导出对象、筛选条件与不包含项；若身份筛选选了老师 / 院领导（导出不支持），弹框会提示该筛选被忽略
- 单个编辑、设置角色等级（1-4 级）

### 2.7 通知群发（信息精准推送）

- **新建群发**：标题 + 正文 + 标签 + 来源
- **接收对象（多选复选框）**：普通学生 / 学生骨干 / 老师 / 院领导，可任意组合
  - 年级 / 专业 / 班级（下拉选择）属于学生属性，**只对学生类对象（普通学生 + 学生骨干）生效**；勾选老师 / 院领导时这些字段对其不起作用，整组接收
  - 只勾老师 / 院领导时，年级 / 专业 / 班级筛选自动隐藏（不适用）
- **预览目标人数**：提交前实时统计命中人数，并**按角色拆分**显示（普通学生 X / 学生骨干 Y / 老师 Z / 院领导 W），便于确认范围
- 渠道可选站内消息（必发）+ 邮件 + 短信模拟
- **24 小时撤回**：广播历史里对已发的通知可撤回，**只删除未读**、已读保留留痕
- **真实邮件**：配置 SMTP 后会真实投递；未配置自动降级为 `email_sim` 站内通知不阻塞业务

### 2.8 操作日志（仅 1 级可见）

- 管理员关键操作流水（通过 / 驳回 / 导入 / 群发 / 设置角色等），按模块和时间筛选
- 通过 `@OperationLog` AOP 切面自动记录，无需手动写入

## 三、微信小程序功能

底部 TabBar 4 个主入口，外加若干二级页面。

### 3.1 首页

- 顶部欢迎卡片显示当前学生姓名 + 学院名
- 未读通知数小红点
- 三快捷入口（政策咨询 / 审批进度 / 党团流程）
- 4 张大卡片（智能问答 / 文件与模板 / 党团进度 / 我的申请）
- "最新通知"显示最近 5 条带标题 + 摘要 + 相对时间

### 3.2 智能问答

- 对话式问答界面，消息气泡（用户右侧 / 系统左侧）
- **三级回答策略**：
  1. 优先匹配知识库标准答案（命中率高时直接返回，含官方链接）
  2. 未命中走 RAG：BGE-small-zh 算向量在 `qa_document_chunk` 检索 → 抽取式回答（带"依据 N"标注 + 来源文档链接）
  3. RAG 也无依据时调 DeepSeek 大模型（v4-flash），AI 回答标"参考建议"
- **每用户每日 200 次提问限流**（按 `qa_chat_log` 表的 user_id + DATE 计数），超过返回错误码 429
- 三个常用问题胶囊预设（政策文档 / 新生报到 / 证明审批）

### 3.3 文件与模板

- 双 Tab 切换：政策文档 / 办公模板
- 文件类型徽标按扩展名配色（PDF 红 / DOC 蓝 / XLS 绿 / PPT 橙 / TXT 灰 / ZIP 紫）
- 类别色块芯片 + 文件大小 + 下载次数
- 点击下载使用 `uni.openDocument` 在微信内嵌打开 PDF / DOCX
- 未上传源文件的模板显示「待上线」徽标

### 3.4 党团进度

- "我的流程"卡片：流程名、当前步骤 / 总步骤、进度条、状态 tag
- "流程模板"展示官方入党 / 入团流程模板入口
- 流程详情页：纵向时间线，每一步显示步骤名、阶段、完成时间或预计天数
- 后端**定时任务**每天 09:00 扫描积极分子等到期步骤，自动给学生发"即将到期" / "已超期"提醒（24 小时防重）

### 3.5 我的申请

- 顶部 3 数字：待审批数 / 可下载数 / 已归档数
- 申请记录列表：模板名作为卡片标题，状态彩色 tag
- 按状态显示不同操作按钮：
  - `pending`：撤回申请
  - `approved`：预览（在线打开 PDF 不锁定）+ 下载证明（确认弹窗后下载并锁定归档）
  - `downloaded`：锁定归档徽标，禁止任何操作
- **提交新申请**流程：选模板 → 个人信息自动从档案带入 → 仅补充模板需要的额外字段（如党员证明需身份证号 / 学历 / 入党日期 / 党支部 / 用途）→ 提交
- **证明 PDF 自动生成**：审批通过后，后端根据模板版式 + 学生信息 + 表单字段，用 PDFBox 嵌入中文字体生成 PDF，学生可预览 / 下载

### 3.6 消息中心

- 横向标签筛选条（从 `/notify/tags` 聚合用户通知里出现过的标签）
- 通知卡片显示标题 + 来源徽章 + 标签 chips + 时间
- 未读条目左侧深红色边框
- 点击条目自动标已读，顶部"全部标记已读"按钮

### 3.7 个人中心

- 头像（CSS 人物剪影） + 姓名 + 学号 + 专业 / 班级
- 三快捷行：我的申请 / 党团进度 / 消息中心
- **联系方式**：邮箱（默认 `学号@ruc.edu.cn`，可点击修改成自定义；清空恢复默认）/ 手机号（含正则校验）
- 我的荣誉列表（由 admin 端录入后同步）
- 退出登录

| 项 | 说明 |
|---|---|
